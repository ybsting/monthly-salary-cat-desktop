# AGENTS.md

## 项目协作规则

本项目是基于 Tauri v2、Vue 3、TypeScript 的桌面宠物应用“月薪喵”。任何自动化助手或开发者在本仓库中修改代码时，都必须遵守以下规则。

## 文档同步规则

每次更新代码，都必须同步更新文档。代码改动完成后，至少检查并按需更新以下文档：

- `README.md`：面向使用者的启动方式、功能说明、配置说明。
- `docs/README.md`：文档入口和模块索引。
- `docs/overview.md`：项目结构、模块职责、功能总览。
- `docs/modules/<module>/implementation.md`：对应模块的实现文档，记录涉及的文件、函数、配置、脚本和运行方式。
- `docs/modules/<module>/principle.md`：对应模块的原理文档，解释功能流程、数据流、关键设计取舍和注意事项。

如果一次代码改动涉及多个模块，需要分别更新所有受影响模块的实现文档和原理文档。常见模块包括：

- `frontend-pet`：桌宠主界面、气泡、右键菜单、工具面板、窗口交互。
- `animation-data`：GIF 资源、动画切换、宠物形象数据。
- `tauri-shell`：Tauri 配置、Rust 后端、托盘、窗口、启动脚本。
- `mail`：邮件发送、收件箱、本地代理、定时邮件、未读邮件提醒。
- `news`：新闻面板、新闻接口、新闻配置。
- `reading`：阅读识别、语音播放。
- `translation`：翻译面板、翻译服务配置。

## 执行要求

- 日常桌面启动必须默认后台静默运行，不依赖前端命令框、Vite 交互终端或任何需要保持打开的控制台窗口；需要本地前端资源时，使用 `scripts/start-background.ps1` 隐藏启动 `scripts/desktop-web-server.cjs` 服务 `dist/`。
- 不允许只改代码不改文档，除非改动完全不影响行为、配置、文件结构、命令或使用方式；即使如此，也应在最终回复中说明“无需更新文档”的原因。
- 文档内容必须和代码保持一致，不写尚未实现的功能。
- 新增脚本、配置、端口、环境变量、启动方式、资源路径时，必须同步写入相关文档。
- 修改用户可感知行为时，必须更新 `README.md` 或对应使用说明。
- 修改模块内部实现时，必须更新对应模块的 `implementation.md` 和 `principle.md`。
- 文档使用中文，说明要具体到文件路径、命令、端口、配置项或函数名，方便后续维护。

## 验证要求

代码改动后应尽量运行可用的验证命令，例如：

- `npm.cmd run build`
- `node --check <script>`
- `cargo check`

如果因为环境限制无法验证，必须在最终回复中明确说明。
